# 系統架構契約（Architecture Contract）

## 這份文件存在的目的

本文件定義此系統的「邊界與核心原則」，用來確保專案在長期演進過程中：

- 不因臨時需求而破壞結構
- 不因技術衝動而增加技術債
- 保持 Public / Private 清晰分離
- 維持索引與原始儲存解耦（Index != Storage）
- 優先使用平台能力（Config over Code）

此文件屬於架構層（Architecture Level），
不是實作細節文件（非 SDD）。

---

## 何時需要參考本文件

在以下情境發生時，必須重新閱讀本文件：

1. 當想新增重大功能（如搜尋、AI Agent、相簿系統）
2. 當考慮改變部署方式或技術框架
3. 當打算自行實作驗證、權限或儲存機制
4. 當系統變得混亂或邊界模糊時
5. 當需要寫 ADR（Architecture Decision Record）時

若新設計違反本文件原則，
應先寫 ADR，說明變更理由與風險，再進行修改。
